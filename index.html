FlowModel




针对大学生的健康管理应用，旨在帮助大学生记录日常健康数据（如睡眠、运动、饮食），并与学校健康中心共享，以便在健康出现风险时提供及时帮助。


流程模型：
- 大学生：通过应用记录健康数据（如睡眠、运动、心情），并接收健康建议。
- 大学健康中心：可以查看学生的健康报告，并在需要时提供支持（如心理咨询、健康建议等）。
- 学业辅导员：在获得学生授权的情况下，可接收健康报告和建议，帮助学生更好地平衡学业与健康。


流程图：
大学生上传数据到健康应用 -> 健康应用自动生成报告并检测异常 -> 异常数据发送给健康中心 -> 健康中心或辅导员提供反馈和帮助建议
HMW（How Might We）问题



1. 我们如何才能帮助学生更方便地记录并管理自己的健康数据？

2. 我们如何才能让学生在健康数据出现异常时收到及时提醒？

3. 我们如何才能为健康中心提供高效的学生健康监测工具？

4. 我们如何才能帮助辅导员更好地了解学生的健康状况，以便提供支持？

5. 我们如何才能提高学生对健康管理的重视，鼓励他们保持健康的生活习惯？

6. 我们如何才能在保证隐私的情况下，让学校更好地支持学生的身心健康？

Personas（人物角色）

Storyboard 1：李健康的健康管理体验



1. 李健康每天早晨打开应用，记录昨晚的睡眠时长和心情。

2. 应用自动检测到他最近几天的睡眠时间较短，显示健康警示。

3. 应用向他推荐了一些改善睡眠的建议，如提前一小时关手机、放松心情。

4. 他尝试应用建议，记录每天的心情和精力变化。

5. 一周后，应用生成报告，显示他的精力和心情有所改善。

6. 健康报告也传送给健康中心，帮助咨询师了解他的健康变化。

7. 王老师联系了李健康，提供额外的心理支持和建议。

8. 李健康感到应用的帮助显著，决定坚持使用。

Storyboard 2：王老师的学生健康监测



1. 王老师在应用中查看学生健康数据，关注高风险学生的情况。

2. 他发现李健康最近的报告显示他的睡眠和情绪波动较大。

3. 王老师通过应用查看李健康授权的健康数据。

4. 他决定联系李健康，安排一个健康咨询时间。

5. 李健康在咨询中得到了舒缓压力的建议。

6. 王老师记录了咨询结果，并在应用中设置了后续关注。

7. 后续应用显示李健康的健康数据逐渐恢复正常。

8. 王老师通过应用反馈，感到这个系统帮助他更好地支持学生。
